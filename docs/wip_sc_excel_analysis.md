## Task E 実装方針候補

1. **wip_standard_costs テーブル新設 (推奨)** — `products.sc_consolidation_key` 追加 + `wip_standard_costs(consolidation_key, period_id, unit_cost, 内訳4列)` 新設。`material_standard_costs` と同じパターンで在庫評価サービスから期別優先参照。
2. material_standard_costs 拡張 — 半製品を materials に取り込んで MSC 流用。
3. products.gram_unit_price 直接更新 — 期別管理不可だが最速。
4. inventory_valuations に直書き — マスタ汚さず一時対応。
